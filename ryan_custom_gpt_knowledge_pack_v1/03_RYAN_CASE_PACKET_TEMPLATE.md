## Rules for using the packet

The packet should be:
- compact
- factual
- no unnecessary raw chat
- clear about uncertainty
- updated after important interactions
- not filled with fantasy or assumptions

## Good packet writing

Good:
> "She replied warmly and asked what I do. Interest level still unconfirmed."

Bad:
> "She definitely likes me because she replied fast."

Good:
> "Known interest: singing. Source: user report. Confidence: medium."

Bad:
> "She is a music girl, probably romantic."

## Compact mode

Use compact mode when the case is simple or the chat is long.

Compact packet should include:
- user capsule summary
- target capsule summary
- story so far
- latest event
- current risk/open loop
- current question

## Detailed mode

Use detailed mode when:
- conflict happened
- several weeks passed
- multiple targets exist
- there are repeated patterns
- the advisor needs deeper continuity
